# Airport TSA Times — Site Map

## Tech Stack
- React 19 + TypeScript + Vite 8
- Tailwind CSS v4 (CSS-based config via `@theme` in index.css)
- React Router v7 (client-side routing)
- Supabase JS client (ready, not yet connected)
- Google Fonts: Inter + JetBrains Mono

## Directory Structure
```
src/
├── main.tsx              # Entry point
├── App.tsx               # Router setup (BrowserRouter)
├��─ index.css             # Tailwind + custom theme + animations
├── lib/
│   ├── types.ts          # All types + utility functions (getWaitLevel, formatTimeAgo)
│   └── supabase.ts       # Supabase client (reads VITE_SUPABASE_URL/KEY from env)

│   ├── WaitTimeBadge.tsx  # Color-coded wait time display (sm/md/lg/xl)
│   ├── AirportCard.tsx    # Grid card: code, city, avg wait, checkpoint preview
│   ├── CheckpointList.tsx # Airport detail: checkpoint rows with trends
│   ├── LiveFeed.tsx       # Real-time report ticker
│   └── TrendChart.tsx     # Hourly bar chart with color-coded bars
└── pages/
    ├── Home.tsx           # Dashboard: search, grid, sort, sidebar with live feed
    ├── AirportDetail.tsx  # Airport page: stats, checkpoints, trends, tips
    └── Report.tsx         # 3-step report flow: airport → checkpoint → time slider
```

## Routes
- `/` — Home dashboard (airport grid + live feed)
- `/airport/:code` — Airport detail (e.g. `/airport/jfk`)
- `/report` — Report wait time (optional `?airport=LAX` preselect)

## Design System
- **Theme**: "Terminal Board" — dark charcoal bg, amber accents, monospace numbers
- **Colors**: terminal-bg (#0a0e17), terminal-card (#111827), amber (#fbbf24)
- **Wait colors**: green (<15m), yellow (15-30m), red (>30m)
- **Typography**: JetBrains Mono for numbers/codes, Inter for body
- **Custom CSS class**: `.board-text` for monospace + letter-spacing

## Data Flow
- Currently using mock data generators (mockData.ts)
- Supabase client initialized but requires env vars to connect
- When connected: airports table, checkpoints table, wait_reports table
- Views for aggregation (current_wait_times)

## Deploy
- Build: `npm run build` → outputs to `dist/`
- Deploy via blrb-connect or GitHub Actions

## Gotchas
- Tailwind v4 uses CSS-based config (`@theme` in index.css), NOT tailwind.config.js
- PostCSS plugin is `@tailwindcss/postcss`, not `tailwindcss`
- Mock data regenerates on component mount (will stabilize with Supabase)
